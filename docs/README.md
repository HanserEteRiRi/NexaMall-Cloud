# NexaMall Cloud（云枢商城）文档

本文档以当前工作树和已执行的验证命令为准。涉及“当前”和“目标”的内容必须严格区分：当前栈可以构建，目标栈代表后续重构方向。

## 阅读顺序

1. [技术栈与版本状态](technology-stack.md)：确认当前版本、目标版本和迁移状态。
2. [架构与模块说明](architecture.md)：了解服务边界、端口、数据库和 Maven 模块组织。
3. [本地开发与构建](development.md)：准备 JDK、基础设施、Nacos 配置并执行构建。
4. [现代化重构路线](modernization/README.md)：查看已完成阶段、后续顺序和验收门槛。

## 历史资料

- `../doc/base`：原始数据库全量 SQL。
- `../doc/properties`：经过脱敏的 Nacos 配置参考，不可直接作为生产配置。
- `../doc/upgrade-*`：旧版数据库增量脚本。
- `../images`：原项目架构图和界面截图。

历史资料会在重构过程中逐步替换为 Flyway、自动化配置模板和可验证部署清单。在替换完成前不要删除，因为它们仍是恢复现有数据结构的重要依据。

## 文档维护规则

- 只有已经落入代码并通过验证的技术才能写入“当前栈”。
- 目标版本升级后必须同步更新版本表、构建命令和验收结果。
- 配置示例只能使用环境变量或明显的占位符。
- 不记录真实域名、IP、账号、Token、Access Key 或密码。
